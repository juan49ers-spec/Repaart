## 3. Fix: Tests fallando en RiderStatsOverview

**Archivo:** `src/features/rider/profile/components/__tests__/RiderStatsOverview.test.tsx`

### Causa raíz
El mock de `lucide-react` no incluía `CheckCircle2` ni `Award`, iconos que el componente usa. Además, el componente había sido refactorizado y el texto `"Progreso Semanal"` había cambiado a solo `"Progreso"`.

### Cambios aplicados

1. **Mock con `importOriginal`** — Reemplazado el mock manual por la API de `importOriginal` que extiende el módulo real:
   ```ts
   vi.mock('lucide-react', async (importOriginal) => {
     const actual = await importOriginal<typeof import('lucide-react')>();
     return { ...actual, /* solo los necesarios */ };
   });
   ```

2. **Iconos añadidos al mock:** `CheckCircle2`, `Award`

3. **Assertions actualizadas:**
   - `"Progreso Semanal"` → `"Progreso"` (refactor del componente)
   - `"vs semana anterior"` → `getByTestId('arrow-up-right-icon')` (texto eliminado del componente)

## 4. Fix: Conexiones Franquicia ↔ Rider

### Problemas encontrados y corregidos

#### #1 y #3 — `franchiseId` incorrecto en modales de rider
**Archivo:** `src/features/rider/home/RiderHomeView.tsx`

**Problema:** Los modales de incidente y checklist usaban `user.franchiseId` (del perfil), ignorando la franquicia del turno activo. Si un rider trabajaba para múltiples franquicias, los incidentes se asociaban siempre a su franquicia de perfil.

**Fix:** Cadena de prioridad: `activeShift.franchiseId → nextShift.franchiseId → user.franchiseId`. Si ninguno tiene franquicia, se muestra un `toast.error()` en lugar de enviar string vacío `''`.

```tsx
const shiftFranchiseId = activeShift?.franchiseId || nextShift?.franchiseId || user.franchiseId;
if (!shiftFranchiseId) {
    toast.error('No se puede reportar: turno sin franquicia asignada.');
    return;
}
```

#### #2 — `user as any` en useRiderSupport
**Archivo:** `src/features/rider/hooks/useRiderSupport.ts`

**Problema:** Se usaba `user as any` para acceder a `franchiseId`, eludiendo el sistema de tipos. El mensaje de error era genérico (`'Usuario no identificado'`).

**Fix:**
- Importado `AuthUser` desde `AuthContext`
- Tipado como `user as AuthUser`
- Mensaje de error mejorado: `'No se puede crear el ticket: el rider no tiene franquicia asignada.'`

#### #4 — Coste de rider con tarifa hardcoded
**Archivos:**
- `src/schemas/users.ts` — añadidos campos
- `src/features/franchise/finance/hooks/useFinancialDataLoad.ts` — refactoring

**Problema:** La nómina estimada de riders se calculaba con constantes hardcoded:
```ts
const ESTIMATED_HOURLY_RATE = 8.5;  // nunca configurable
const SOCIAL_SECURITY_RATE = 0.32;  // nunca configurable
```

**Fix:**
- Añadidos campos `riderHourlyRate` y `socialSecurityRate` al schema de usuario (`UserSchema` en `src/schemas/users.ts`)
- El perfil de franquicia se carga ahora dentro del `Promise.all` (en paralelo con los demás datos)
- Las tarifas hardcoded pasan a ser fallbacks:
```ts
const hourlyRate = profile?.riderHourlyRate ?? DEFAULT_HOURLY_RATE;  // 8.5 €/h
const ssRate = profile?.socialSecurityRate ?? DEFAULT_SS_RATE;        // 32%
```
- Eliminado el bloque `let profile` duplicado que existía después del cálculo

#### #5 — Componente muerto `CoverageStats.tsx`
**Archivo:** `src/features/franchise/dashboard/widgets/CoverageStats.tsx`

**Problema:** El componente estaba definido y exportado pero nunca importado ni renderizado en ningún lugar del proyecto. Usaba `any[]` como tipo para sus props principales.

**Fix:** Eliminado el archivo.
